# Phishing / Scam Text Detector – UI & Business Logic

## Overview

Сервис решает один основной сценарий: пользователь вставляет текст подозрительного сообщения и получает понятный вердикт о риске фишинга/скама.[web:35]

Интерфейс включает:

- Стартовый экран с полем для текста и кнопкой запуска проверки.
- Экран результата с вердиктом, оценкой риска и краткими пояснениями.
- Разделы с информацией об угрозах и принципах работы сервиса простым языком.[web:26]
- Личный кабинет с историей проверок.

---

## Основные экраны

### 1. Экран проверки текста

- Поле для ввода/вставки текста.
- Кнопка запуска анализа.
- Минимальные настройки (например, тип сообщения).

### 2. Экран результата

- Вердикт по уровню риска (высокий, средний, низкий).[web:35]  
- Числовая оценка риска и цветовая индикация.
- Подсветка важных участков текста (наиболее существенных для решения).[web:9]
- Краткий блок “почему так” с ключевыми признаками и подозрительными элементами.
- Рекомендации по дальнейшим действиям (не переходить по ссылкам, перепроверить отправителя и т.п.).[web:26]

### 3. Экран истории

- Список прошлых проверок с датой, кратким описанием и итогом.
- Возможность открыть подробный результат с подсветкой.

### 4. Информационные страницы

- Объяснение, что такое фишинг/скам и типичные признаки.[web:26]
- Общее описание логики проверки без технических деталей.

---

## Действия пользователя

Пользователь может:

- Ввести или вставить текст и отправить его на анализ.
- Зарегистрироваться и войти в личный кабинет.
- Получить и просмотреть результат с вердиктом, оценкой риска и подсвеченными фрагментами.[web:35][web:9]
- Наводить курсор или кликать по подсвеченным участкам, чтобы видеть пояснения.[web:9]
- Переключать режим подсветки (по модели, по правилам или комбинированный).
- Менять текст и перезапускать проверку, чтобы видеть изменение оценки риска.
- При наличии аккаунта — просматривать историю проверок.

---

## Общая архитектурная идея

Архитектурно это веб‑сервис, в котором:

- Веб‑интерфейс отправляет текст на анализ и отображает результат (вердикт, подсветка, рекомендации).[web:24]
- Сервис анализа принимает текст, последовательно обрабатывает его, оценивает риск и применяет набор правил, формируя единый результат.[web:26]
- Слой хранения данных отвечает за историю проверок, конфигурации и служебную информацию.[web:24]
